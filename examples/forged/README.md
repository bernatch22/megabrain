# Forged strategies — real `megabrain forge --specialize` output

These are **verbatim outputs** of `megabrain forge --specialize` on real repos —
not hand-written. Each was generated by an LLM from the repo's own files and
installed only after **winning a measured retrieval A/B** against the built-in
chunker (`forge_eval.ab_gate`): neutral probe spans derived from each file's own
structure, both variants indexed for real, span-IoU + hit@k scored on every file
the candidate changes. A valid-but-worse chunker is measured to be worse and
rejected.

## Why specialization exists

The built-in chunkers merge small syntax units up to a 4000-char budget for
context-rich embeddings. That's good for most code, but a file that is one big
**data table** or a class of **many short methods** collapses into a single
blob — so a query about *one* entry retrieves the *whole* file. Specialization
re-chunks exactly those files and delegates everything else unchanged
(shape-router).

## [`sinatra_ruby_rb.py`](sinatra_ruby_rb.py) — Ruby, sinatra

```
megabrain forge --specialize ~/sinatra --ext .rb
```

`detect_specialization` flagged 12 `.rb` files (target
`sinatra-contrib/lib/sinatra/cookies.rb`, 345 lines, 85% in one chunk). One LLM
call produced a shape-router: `_is_special` fires on files with >10 short method
defs; `_cut_lines` cuts every ~500 non-whitespace chars, snapping to `def`
lines; everything else delegates to the built-in Ruby chunker.

**Gate: WIN** — pooled span-IoU **0.049 → 0.171 (3.5×)** across the **15** files
it changed, **every one improved**, worst-file +0.064 (no regression):

| file | IoU built-in → forged |
|---|---|
| test/contest.rb | 0.071 → 0.322 |
| sinatra-contrib/lib/sinatra/streaming.rb | 0.027 → 0.155 |
| rack-protection/lib/rack/protection/base.rb | 0.024 → 0.153 |
| sinatra-contrib/lib/sinatra/cookies.rb | 0.039 → 0.103 |
| … 11 more, all positive | |

### The win, concretely

Query: *"how does the cookies helper delete a cookie by key"* — `delete()` lives
at lines 113–118 of `cookies.rb`.

| | top retrieved chunk of the file |
|---|---|
| **built-in** | **L4–304 (301 lines)** — the entire `Jar` class as one blob, all 39 methods |
| **forged** | **L89–131 (43 lines)** — a tight window that actually contains `delete()` |

## Also verified

- **`.py` / psf/requests** — `status_codes.py` (a 68-entry HTTP-status dict left
  as one blob): span-IoU **0.009 → 0.132 (14×)**, hit@1 0.50 → 0.71, every other
  `.py` file byte-identical. The forged router beat a hand-written reference
  (0.098) under the same gate.
- **`.js` / expressjs/express** — blob-shaped `lib/*.js`: pooled IoU
  0.040 → 0.092, 5 files improved, worst-file +0.001.

## Reproduce

```bash
megabrain forge --specialize <repo> --list      # free census: what's poorly chunked
megabrain forge --specialize <repo> --dry-run    # generate + gate, print code, don't install
megabrain forge --specialize <repo>              # install the winners; auto-loaded thereafter
```
